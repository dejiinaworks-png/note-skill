# note バズ投稿自動化システム 要件定義書

**バージョン:** 1.0.0
**作成日:** 2026-03-08
**ステータス:** Draft

---

## 0. エグゼクティブサマリー

Claude Code上で動作する「noteバズ投稿自動生成スキル」を構築する。
ユーザーがテーマを入力するだけで、**リサーチ → 企画 → 文章生成 → 図解作成（NanoBanana2）→ バナー生成 → note投稿**まで一貫して自動化するシステム。
最終的にはClaudeスキルとして展開・配布可能な形に仕上げる。

---

## 1. プロジェクト概要

### 1.1 目的

| 項目 | 内容 |
|------|------|
| 主目的 | noteでバズる記事を効率的に量産する |
| 副目的 | スキル化して他ユーザーへ展開 |
| ターゲット読者 | noteで発信したいクリエイター・起業家・フリーランス |
| KPI | スキ数50以上 / PV 1,000以上 / フォロワー増加 |

### 1.2 システム名

```
note-skill（note バズ投稿自動化スキル）
```

---

## 2. システム全体フロー

```
[ユーザー入力]
    ↓ テーマ・キーワード・ターゲット読者
[Phase 1: ディープリサーチ]
    ↓ システムリサーチ専用コピペ.md の手順に準拠
    ↓ note/X/YouTube/Reddit/論文等から情報収集
[Phase 2: 企画・構成設計]
    ↓ バズる記事タイトル候補3案生成
    ↓ 見出し構成（H2/H3）設計
    ↓ 図解挿入箇所の設計
[Phase 3: 文章生成（Claude）]
    ↓ 本文執筆（読者の悩み解決型・SEO最適化）
    ↓ 各セクションに合わせた文体・長さ調整
[Phase 4: 図解生成（NanoBanana2）]
    ↓ 図解プロンプト自動生成
    ↓ NanoBanana2 で図解・インフォグラフィック作成
    ↓ バナー画像（タイトル入り）作成
[Phase 5: レイアウト組み立て]
    ↓ 文章 + 図解を最適位置に配置
    ↓ note markdown形式に整形
[Phase 6: 品質チェック]
    ↓ 読みやすさスコア評価
    ↓ バズ要素チェックリスト確認
[Phase 7: note投稿]
    ↓ note API または手動投稿用フォーマット出力
[ログ・issue記録]
    ↓ GitHub issueに投稿結果・分析を自動記録
```

## 3. 機能要件

### 3.1 Phase 1 - ディープリサーチ機能（2段階方式）

**設計方針（Windows用リサーチ提案.md 準拠）**
- **Pass1（広範リサーチ）**: seed_urlsを起点にサイト構造を把握し、国内・海外・全SNSをリサーチ
- **Pass2（深掘りリサーチ）**: Pass1の未確定事項・ギャップを補完し新たな発見を抽出
- **フェーズ境界で /compact 実行**: 重要事項のみ保持しコンテキスト圧迫を防止
- **重要主張には出典URL・確認日を必ず記載**
- **全文転載禁止**: 要点抽出・引用のみ

| 機能ID | 機能名 | 詳細 |
|--------|--------|------|
| F1-01 | キーワード拡張 | 入力テーマから関連・複合・急上昇・ニッチキーワードを自動抽出（keyword_universe.csv形式） |
| F1-02 | note記事リサーチ | noteの人気記事・トレンド・カテゴリ体系を分析 |
| F1-03 | X（旧Twitter）リサーチ | 関連ツイートの傾向・エンゲージメント・議論クラスタを分析 |
| F1-04 | YouTube文字起こし分析 | 関連動画の文字起こしから論点・需要・ペインを抽出 |
| F1-05 | 海外情報リサーチ | Reddit・英語圏メディア・論文からグローバルトレンド収集 |
| F1-06 | Pass1 成果物生成 | `research/source_index.md`・`research/market_maps.md`・`research/social_signals.md` を保存 |
| F1-07 | Pass2 深掘りリサーチ | Pass1の穴埋め・反証・見落とし確認・新規発見に集中した2回目リサーチ |
| F1-08 | Pass2 成果物生成 | `research/pass2_deltas.md`（差分）・`research/gaps_closed.md`（ギャップ潰し証跡）を保存 |
| F1-09 | 専門分野深掘り | テーマに専門分野があれば専門リサーチサイトまで深掘り |

### 3.2 Phase 2 - 企画・構成設計機能

| 機能ID | 機能名 | 詳細 |
|--------|--------|------|
| F2-01 | タイトル3案生成 | バズタイトルのパターン（数字型・問いかけ型・衝撃型）で3案生成 |
| F2-02 | 記事構成設計 | H2/H3見出し構成を自動設計（2,000〜5,000字想定） |
| F2-03 | 図解挿入計画 | どのセクションに何の図解を入れるか設計 |
| F2-04 | CTA設計 | フォロー・スキ・コメント誘導のCTAを各記事に最適化 |
| F2-05 | ペルソナ設定 | ターゲット読者のペルソナを明確化し文章トーンに反映 |

### 3.3 Phase 3 - 文章生成機能（Claude）

| 機能ID | 機能名 | 詳細 |
|--------|--------|------|
| F3-01 | リード文生成 | 読者を引き込む冒頭300字の自動生成 |
| F3-02 | 本文執筆 | 各H2セクションを構成に沿って執筆 |
| F3-03 | 読みやすさ制御 | 1段落3〜4文・箇条書き活用・体言止め禁止等のルール適用 |
| F3-04 | SEO最適化 | キーワード自然挿入・見出しへのキーワード配置 |
| F3-05 | まとめ・CTA生成 | 記事末尾のまとめとフォロー誘導文の自動生成 |

| 機能ID | 機能名 | 詳細 |
|--------|--------|------|

## 4. 非機能要件

### 4.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| 記事1本の生成時間 | 10分以内（リサーチ込み） |
| 図解生成時間 | 1枚あたり2分以内 |
| コンテキスト管理 | /compact を適切に活用し長時間作業に対応 |

### 4.2 品質基準

| 項目 | 基準 |
|------|------|
| 記事文字数 | 2,000〜5,000字 |
| 図解枚数 | 記事あたり3〜5枚 |
| バズスコア | 70点以上で投稿推奨 |
| オリジナリティ | リサーチ情報に独自解釈・事例を追加 |

### 4.3 スキル化要件

| 項目 | 詳細 |
|------|------|
| 独立性 | 外部依存を最小化し単体で動作可能 |
| 設定可能性 | ターゲット・文体・テーマをパラメータで変更可能 |
| 再現性 | 同じ入力で品質の安定した出力を保証 |
| ドキュメント | README・使い方ガイドを同梱 |
| 配布形式 | `.claude/skills/` に配置するMarkdownスキルファイル |

## 6. バズ記事の設計原則

### 6.1 noteでバズるタイトルパターン

| パターン | 例 |
|---------|-----|
| 数字型 | 「AIで月収10万を達成した5つの方法」 |
| 問いかけ型 | 「なぜあなたのnote記事は読まれないのか？」 |
| 衝撃型 | 「ChatGPTより優秀。Claudeでブログを量産した結果」 |
| ノウハウ型 | 「初心者でも3日でできるnote収益化の全手順」 |
| 体験談型 | 「副業0円→月5万円になった私のnote戦略」 |

### 6.2 読者を飽きさせない構成原則

```
[リード文] 読者の悩みに共感 → 解決の予告
    ↓
[H2-1] 問題提起 → 図解①（現状の問題を可視化）
    ↓
[H2-2] 解決策の概要 → 図解②（解決フロー）
    ↓
[H2-3] 具体的手順STEP1 → （テキスト主体）
    ↓
[H2-4] 具体的手順STEP2 → 図解③（手順の比較・整理）
    ↓
[H2-5] 応用・事例 → （テキスト主体）
    ↓
[まとめ] 要点整理 → 図解④（まとめインフォグラフィック）
    ↓
[CTA] フォロー・スキ・コメント誘導
```

### 6.3 図解の役割分担

| 図解種別 | 配置場所 | 目的 |
|---------|---------|------|
| バナー | 記事冒頭（アイキャッチ） | 第一印象・クリック率向上 |
| フロー図 | H2-2（解決策概要後） | 全体像の把握 |
| 比較図 | H2-3〜4（手順説明中） | 具体的な手順の視覚化 |
| まとめ図 | 記事末尾 | 読了後の記憶定着・シェア促進 |

## 7. スキル化仕様

### 7.1 スキルの起動方法

```bash
# ユーザーはこれだけ入力すればOK
/note-post テーマ:"AIで副業" ターゲット:"20代サラリーマン" 文体:"語りかけ調"
```

### 7.2 スキルの出力物

| 出力物 | 形式 | 保存先 |
|--------|------|--------|
| 完成記事 | Markdown | `output/articles/YYYYMMDD_タイトル.md` |
| バナー画像 | PNG (1280×670) | `output/images/YYYYMMDD_banner.png` |
| 図解画像 | PNG | `output/images/YYYYMMDD_fig_01〜04.png` |
| リサーチレポート | Markdown | `output/articles/YYYYMMDD_research.md` |
| 投稿チェックリスト | Markdown | ターミナル出力 |
| GitHub issue | - | dejiinaworks-png/note-skill に自動作成 |

### 7.3 スキル配布パッケージ構成

```
note-buzz-skill-v1.0.zip
├── README.md           # インストール手順・使い方
├── CHANGELOG.md        # バージョン履歴
├── .claude/skills/     # スキルファイル本体
├── .claude/agents/     # 専門エージェント群
├── templates/          # テンプレート集
└── .mcp.json           # MCP設定サンプル
```

## 9. 制約・リスク管理

| リスク | 対策 |
|--------|------|
| コンテキスト超過 | /compact を適切なタイミングで実行。長いリサーチはエージェント分割 |
| NanoBanana2 の仕様変更 | プロンプトテンプレートを抽象化し変更容易な設計にする |

| 生成コンテンツの品質バラツキ | バズスコア70点未満は自動的に改善ループ実施 |

## 付録 A: バズスコア評価基準

| 評価項目 | 配点 | 評価基準 |
|---------|------|---------|
| タイトルの魅力度 | 20点 | 数字・問いかけ・衝撃ワードの含有 |
| リード文の引力 | 15点 | 読者の悩みへの共感度 |
| 情報の希少性 | 15点 | 他記事にない独自情報・視点 |
| 読みやすさ | 15点 | 段落構成・漢字率・文長の適切さ |
| 図解の質と量 | 15点 | 3枚以上・内容との一致度 |
| SEO対応 | 10点 | キーワード配置・見出し最適化 |
| CTA設計 | 10点 | フォロー・スキ誘導の自然さ |
| **合計** | **100点** | **70点以上で投稿推奨** |

## 付録 B: NanoBanana2 プロンプトテンプレート

### バナー生成
```
タイトル: [記事タイトル]
スタイル: noteアイキャッチバナー
サイズ: 1280×670px
デザイン: モダン・読みやすい・インパクトあり
カラー: [テーマカラー]
テキスト配置: 中央
背景: グラデーション or イラスト背景
```

### フロー図生成
```
種別: フローチャート
テーマ: [図解内容]
ステップ数: [N]ステップ
各ステップ: [STEP1の内容] → [STEP2の内容] → ...
スタイル: シンプル・矢印あり
カラー: ブルー系 or [テーマカラー]
```

### まとめインフォグラフィック
```
種別: まとめインフォグラフィック
タイトル: [記事のまとめタイトル]
要点数: [N]個
各要点: [要点1] / [要点2] / ...
スタイル: アイコン付き・カード型
カラー: [テーマカラー]
```
