# 論理的整合性チェックガイド

## 数字設定の整合性ルール

### 冒頭で設定した基準の維持
冒頭で「月商300万円の壁を超えられない」と設定した場合：

**OK例**
- 事例：「月商100万円で停滞していた人」
- 事例：「月商200万円から伸び悩んでいた人」
- 事例：「月商280万円で頭打ちになった人」

**NG例**
- 事例：「月商500万円で停滞していた人」（基準を上回っている）
- 事例：「月商400万円で悩んでいた人」（基準を上回っている）

### 例外的に基準を上回る場合の処理方法

**1. 明確な理由提示パターン**
```
「月商300万円の壁を超えられない人が87%」と設定後に、
「月商500万円で停滞」の事例を使う場合：

「一度は300万を超えたものの、500万円で完全に頭打ち」
「300万円を超えてからも、次の壁で行き詰まった」
```

**2. 段階的成長の説明パターン**
```
「彼は300万円の壁は越えたものの、その後の500万円の壁で2年間停滞していました」
```

### 業界・カテゴリ別基準の設定

**高額商品系**
- 基準：「月商1000万円の壁」
- 事例：300万円〜800万円で停滞

**低価格商品系**
- 基準：「月商100万円の壁」
- 事例：30万円〜80万円で停滞

**中価格商品系**
- 基準：「月商300万円の壁」
- 事例：50万円〜200万円で停滞

## 整合性チェックポイント

### 冒頭設定チェック
□ 冒頭で設定した数字・条件を明確に記録
□ 対象読者の現状レベルを把握
□ 提示する壁・問題のレベルを確定

### 事例作成時チェック
□ 事例の数字が冒頭設定を下回っているか
□ 上回る場合は明確な理由があるか
□ 読者が違和感を持たない流れになっているか

### 全体通読チェック
□ 冒頭から結論まで数字の矛盾がないか
□ 読者視点で「おかしい」と思う箇所がないか
□ ストーリーの流れが自然で説得力があるか

## よくある整合性エラーパターン

### パターン1：壁設定の矛盾
```
NG: 「月商100万円の壁を越えられない」
    → 事例「300万円で停滞」

OK: 「月商100万円の壁を越えられない」
    → 事例「80万円で停滞」
```

### パターン2：成功レベルの矛盾
```
NG: 「初心者が稼げない」
    → 事例「年収1000万円の人が悩んでいた」

OK: 「初心者が稼げない」
    → 事例「副業で月10万円も稼げなかった人」
```

### パターン3：時系列の矛盾
```
NG: 「今まで結果が出なかった」
    → 「実は以前から月商500万円」

OK: 「今まで結果が出なかった」
    → 「月商30万円から伸び悩んでいた」
```

## 修正方法

### 事例の数字調整
- 冒頭設定以下の数字に変更
- 複数事例がある場合は段階的に配置

### 理由説明の追加
- なぜその数字になったかの経緯を説明
- 一時的成功からの転落パターンで説明

### 設定の見直し
- 冒頭の基準設定自体を調整
- より適切なレベルに設定変更

このガイドに従うことで、読者が「あれ？おかしい」と感じる論理的矛盾を防ぎ、説得力のあるコラムが作成できます。
