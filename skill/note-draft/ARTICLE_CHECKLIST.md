# 記事品質チェックリスト v2.0

qa-lead エージェントがこのチェックリストを使って最終レビューを行う。

---

## A. 構成チェック

| # | 項目 | OK | NG例 |
|---|------|----|------|
| A1 | frontmatter に title と tags（5個）がある | | タグが4個以下 |
| A2 | 冒頭フックが個人体験・共感で始まる | | 「この記事では〜を解説します」 |
| A3 | H2 大見出しが3〜5個ある | | H2が2個以下 or 6個以上 |
| A4 | 各H2に H3 小見出しが2個以上ある | | H3がないH2がある |
| A5 | 記事末尾に「まとめ」セクションがある | | まとめなしで終わっている |

## B. フォーマットチェック

| # | 項目 | OK | NG例 |
|---|------|----|------|
| B1 | 箇条書き（`- `）がリスト表現に使われている | | 箇条書きがなく文章のみ |
| B2 | コードブロック（` ``` `）がコマンドに使われている | | コマンドが地文に混じっている |
| B3 | 引用ブロック（`> `）が要件リスト or URLまとめに使われている | | 引用ブロックが一切ない |
| B4 | 太字（`**text**`）が重要箇所に使われている | | 太字が全くない |
| B5 | 図解プレースホルダーが本文中に2枚以上ある | | 図解が1枚 or なし |
| B6 | `---` セクション区切りが適切に使われている | | 区切りなしで詰まっている |

## C. 内容品質チェック

| # | 項目 | OK | NG例 |
|---|------|----|------|

## D. 初心者向け必須情報チェック

**ソフトウェア/ツールを紹介する記事で必須:**

| # | 項目 | OK | NG例 |
|---|------|----|------|
| D1 | インストール先の公式DLリンクがある | | 「インストールしてください」だけでURLなし |
| D2 | アカウント登録手順・登録URLがある | | アカウントが必要なのに登録方法の説明なし |
| D3 | 必要なもの（前提条件）が記事上部にリスト化されている | | 途中で「実はNode.jsが必要でした」 |
| D4 | Windows/Mac 両対応の記事は両方の手順が書かれている | | Macだけ or Windowsだけ |
| D5 | バージョン要件が明記されている | | 「最新版をインストール」だけ |

## E. リンク・参照チェック

| # | 項目 | OK | NG例 |
|---|------|----|------|
| E1 | 公式サイトURLが全てフルURL（https://）で記載されている | | テキストのみでURLなし |
| E2 | 参考リンクが記事末尾にまとめられている | | リンクが本文中にバラバラ |

---

## 判定基準

- **A〜E 全項目 OK** → Phase 6（投稿）へ進む
- **1つでも NG** → sub-implementer に修正を依頼してから再チェック
- **D1〜D5 のいずれかが NG** → 必ず修正（初心者が迷子になる最大の原因）
