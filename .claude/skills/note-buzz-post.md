# note バズ投稿自動生成スキル v2.0

## スキル概要

noteでバズる記事を自動生成・投稿するスキル。
「AIが書いたとバレない」品質を最優先に設計。

## 起動方法

```
/note-post テーマ:"[テーマ]" ターゲット:"[読者層]" 文体:"[文体]"
```

---

## 絶対ルール: AI臭を消す

### やってはいけないこと（違反したら書き直し）
- 「〜と言えるでしょう」「〜ではないでしょうか」を使う
- 「これにより」「このように」で段落を始める
- 「多くの〜が」「効果が期待されます」など抽象表現
- 「です。」「ます。」が3回以上連続する
- 同じ内容を言い換えて繰り返す（同義反復）
- 文の長さが全部同じ（リズムがない）
- フォーマルすぎる丁寧語
- 箇条書きの乱用（3連続以上禁止）

### やるべきこと
- 語尾を意図的にばらす: です/だ/体言止め/口語/疑問形/倒置
- 一文に長短のリズムをつける（短い文→長い文→短い文）
- 「ぶっちゃけ」「正直」「マジで」など口語を自然に混ぜる
- 失敗談・未完成の途中経過を入れる（"不完全さ"が人間の証拠）
- 具体的な固有名詞・数値を入れる
- 体験ベースで書く（「試してみたら」「やってみたところ」）
- 感情を入れる（「驚いた」「焦った」「嬉しかった」）

---

## Phase 1: ディープリサーチ（3〜5分）

3つのエージェントを並列起動:
1. **テーマの最新情報収集**: WebSearch で関連ニュース・データ・事例
2. **note.comバズ記事分析**: 同テーマでスキが多い記事のタイトル・構成を調査
3. **読者ペイン調査**: ターゲット読者が何に悩んでいるか

結果を統合して記事企画に反映。

---

## Phase 2: 企画・構成設計（2分）

### タイトル設計（3案作成→1つ選択）

**効果的なパターン**:
- 数字型: 「〇〇する△つの方法」
- 問いかけ型: 「なぜ〇〇は△△なのか？」
- 自己開示型: 「正直に言うと、〇〇を知らなかった」
- 衝撃型: 「〇〇した結果がやばすぎた」

**タイトルルール**:
- 30文字以内
- 【2026年最新】などの鮮度タグは入れてもいいが、乱用しない
- 「完全」「徹底」は使いすぎると逆にAI臭い

### ハッシュタグ設計（必須5タグ）

リサーチ結果から以下の順序で5タグを決定する。手動で適当に決めてはいけない。

| 優先順位 | ルール | 例（Claude Opus 4.6の記事） |
|---------|--------|--------------------------|
| 1位（必須）| **主題の固有名詞をそのまま**。ブランド名・製品名・サービス名 | `Claude` |
| 2位（必須）| **サブ固有名詞**。会社名・バージョン・関連ツール | `Anthropic` or `生成AI` |
| 3位 | リサーチで頻出したnote.com検索ワード（note-research結果から抜粋） | `AI活用` |
| 4位 | 記事の問題カテゴリ（職種・ジャンル・悩み系） | `エンジニア` |
| 5位 | 競合・比較ワード（比較記事の場合のみ。なければジャンル補完） | `LLM` |

**禁止ルール**:
- 記事の主題が競合製品でないのに競合製品名をタグにしない（例: Claude記事に`ChatGPT`は不可）
- `AI`単体など汎用すぎるタグだけで埋めない
- リサーチで一度も出てこなかった造語タグは使わない

**確認チェック**:
- [ ] タグ1に記事の主役（製品名・人名・サービス名）が入っているか
- [ ] 5タグ中3タグ以上がnote-researchで実際に確認したキーワードか

---

### 参考URL記載ルール

参考リンクを記事末に載せる場合、**必ずリサーチ中に実際にアクセス確認したURLのみ**使用する。

- WebFetch等でアクセスして200 OKを確認したURLのみ記載
- AIが推測で生成したURL（「おそらくこのURLだろう」）は絶対に載せない
- 確認できるURLがない場合は参考セクションごと削除する

**文末ハッシュタグ（`#AI #Claude ...`形式）は本文末に記載しない。**
タグはフロントマター（`tags:`セクション）にのみ書く。note.comへの投稿時にタグ欄として設定される。

### 記事構成

```
[サムネイル画像]

冒頭（リード文）: 3〜5文
  - 読者の悩みへの共感 or 意外な事実 or 体験の切り出し
  - 「この記事を読むと何が得られるか」を1文で

---

H2-1（問題提起 or 導入）: 300〜500字
  [図解①: 現状の問題を可視化]

H2-2（解決策 or 本題）: 400〜600字
  [図解②: フローチャート or 比較図]

H2-3（具体例 or 深掘り）: 400〜600字

H2-4（注意点 or 別視点）: 200〜400字
  [図解③: まとめ図]

---

まとめ: 200〜300字
  - 要点を3行でまとめる
  - 読者への問いかけで締める

おわりに: 100〜150字
  - 「スキ」「フォロー」の自然な誘導
  - コメント誘導（「あなたの経験も教えてください」）
```

**文字数目安**: 1,500〜2,500字（通勤中に読み切れる長さ）

## Phase 3: 文章生成（3〜5分）

### リード文テンプレート（5パターンから選択）

**パターンA: 自己開示型**
```
正直に言うと、3ヶ月前の自分は[テーマ]のことを何も知らなかった。
[きっかけの一文]
今は[変化後の状態]。その過程を全部書く。
```

**パターンB: 問いかけ型**
```
[読者の悩み]って思ったこと、ありませんか？
僕も同じだった。[共感の一文]
でも[転機]をきっかけに、考え方が変わった。
```

**パターンC: 数字インパクト型**
```
[具体的な数字の成果]。
嘘みたいだけど、本当の話。
[背景の一文]を共有したい。
```

**パターンD: 意外性型**
```
[常識の逆を行く主張]。
最初に聞いたとき、正直「は？」と思った。
でも調べてみたら、[根拠]だった。
```

**パターンE: 体験切り出し型**
```
先週、[具体的なエピソード]があった。
[結果や感想]。
これ、[テーマ]を考える上でめちゃくちゃ大事な話だと思う。
```

### 語尾バリエーション（自動チェック）

3文連続で同じ語尾を使わない。以下をローテーション:
- です / ます
- だ / である
- 体言止め（「〜という話。」「〜なのだ。」）
- 口語（「〜なんですよね」「〜だと思う」）
- 疑問形（「〜じゃないだろうか？」）
- 倒置（「驚いた、正直。」）

## Phase 4: 画像生成（1〜2分）

### サムネイル（1280x670px）
Gemini API (gemini-3.1-flash-image-preview) で生成。

**サムネイルルール**:
- タイトルの核心ワードを大きく配置
- 色はメインカラー1色 + アクセント1色
- 中央に重要要素（端はトリミングされる）
- テキストは読める大きさ

### 記事内図解（3〜4枚）

Gemini API で生成し、Markdown内に `![alt](path)` で配置。
publish-article.js が自動検出して挿入。

**配置ルール**: 見出し（H2）の直下に1枚ずつ

| 位置 | 図解の種類 | 目的 |
|------|-----------|------|
| H2-1の下 | 問題の可視化（比較図） | 読者の「あるある」を視覚化 |
| H2-2の下 | フローチャート | 解決策の全体像を一目で |
| H2-4の下 | まとめ図 | 要点整理で記憶に残す |

---

## Phase 5: 自動投稿

```bash
node scripts/publish-article.js output/articles/[記事].md output/articles/thumbnail.png publish
```

### Markdownファイルフォーマット
```markdown
---
title: 記事タイトル（30字以内）
tags:
  - タグ1
  - タグ2
---

リード文...

## 見出し1

本文...

![図解の説明](images/figure1.png)

## 見出し2

本文...
```

`![alt](path)` の画像マーカーを publish-article.js が検出し、
エディター内で自動挿入。

## Phase 6: 品質チェックリスト

投稿前に以下を全て確認:

- [ ] タイトルは30文字以内か
- [ ] リード文に「共感」「ベネフィット」が含まれるか
- [ ] 「〜と言えるでしょう」を使っていないか
- [ ] 同じ語尾が3回連続していないか
- [ ] 具体的な固有名詞・数値が入っているか
- [ ] 体験ベースのエピソードが1つ以上あるか
- [ ] 文字数は1,500〜2,500字か
- [ ] 画像は見出し直下に3枚以上入っているか
- [ ] CTAは押しつけがましくないか
- [ ] 通読して「AI臭」を感じないか
