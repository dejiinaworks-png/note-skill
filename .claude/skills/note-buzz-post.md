# note バズ投稿自動生成スキル

## スキル概要

noteでバズる記事を自動生成するスキル。テーマを入力するだけで、リサーチ→企画→文章→図解（NanoBanana2）→投稿フォーマットまで一気通貫で出力する。

## 起動方法

```
/note-post テーマ:"[テーマ]" ターゲット:"[読者層]" 文体:"[ですます/だである/語りかけ]"
```

### 例
```
/note-post テーマ:"Claude Codeで副業" ターゲット:"30代会社員" 文体:"語りかけ調"
```

## 実行フロー

### Phase 1: ディープリサーチ（2段階）

**Pass1 - 広範リサーチ**
以下を検索・分析してリサーチレポートを作成:
- note / X / YouTube / Reddit で関連情報を収集
- 関連キーワード（関連・複合・急上昇・ニッチ）を抽出
- 競合記事の構成・反響を分析

**Pass2 - 深掘りリサーチ**
- Pass1の未確定事項・ギャップを補完
- 独自視点・差別化ポイントを発掘
- 読者ペインの根拠データを収集

リサーチ結果を `output/articles/[YYYYMMDD]_research.md` に保存。

### Phase 2: 企画・構成設計

以下を設計してユーザーに提示・確認:

**タイトル案 3本**（パターン別）
- 数字型: 「〇〇する△つの方法」
- 問いかけ型: 「なぜ〇〇は△△なのか？」
- 衝撃型: 「〇〇した結果がやばすぎた」

**記事構成（H2/H3）**
- リード文
- H2-1〜H2-4（各セクション）
- まとめ + CTA

**図解設計**
- バナー（アイキャッチ）
- フロー図（解決策の概要）
- 比較図（手順説明）
- まとめ図（要点整理）

### Phase 3: 文章生成（Claude）

構成に沿って本文を執筆:

**執筆ルール**
- 1段落 3〜4文
- 箇条書きは3項目以上のとき使用
- 1,000字ごとに視覚的区切りを設置
- 読者への語りかけを意識（「あなたも〜」「実は〜」）
- SEOキーワードを見出しと冒頭に自然に挿入
- 文字数: 2,000〜5,000字

**リード文の型**
```
[読者の悩みへの共感（2〜3文）]
[解決できることの宣言（1〜2文）]
[この記事で得られること（箇条書き3項目）]
```

### Phase 4: NanoBanana2 図解生成

各図解のプロンプトを生成して出力:

**バナー（アイキャッチ）**
```
[NanoBanana2プロンプト]
種別: noteアイキャッチバナー
タイトル: [記事タイトル]
サイズ: 1280×670px
スタイル: モダン・インパクト重視
カラー: [テーマカラー]
テキスト: 中央配置・大きめフォント
```

**フロー図**
```
[NanoBanana2プロンプト]
種別: フローチャート
内容: [記事の解決策フロー]
ステップ数: [N]
スタイル: シンプル・矢印あり・カラー背景
```

**比較図**
```
[NanoBanana2プロンプト]
種別: 比較表 or Before/After
内容: [比較内容]
スタイル: 2カラム・アイコン付き
```

**まとめ図**
```
[NanoBanana2プロンプト]
種別: まとめインフォグラフィック
要点: [N]個
スタイル: カード型・番号付き
```

### Phase 6: 品質チェック

**バズスコア評価（100点満点）**

| 項目 | 配点 | チェック内容 |
|------|------|------------|
| タイトル魅力度 | 20点 | 数字・問いかけ・衝撃ワード含有 |
| リード文引力 | 15点 | 読者の悩みへの共感度 |
| 情報の希少性 | 15点 | 独自情報・視点の有無 |
| 読みやすさ | 15点 | 段落・漢字率・文長 |
| 図解の質と量 | 15点 | 3枚以上・内容一致 |
| SEO対応 | 10点 | キーワード配置 |
| CTA設計 | 10点 | フォロー・スキ誘導 |

**70点未満 → 自動改善ループ実施**
**70点以上 → 投稿推奨**

### Phase 7: 出力・記録

**出力ファイル**
- `output/articles/[YYYYMMDD]_[タイトル].md` — 完成記事
- `output/articles/[YYYYMMDD]_nanobana_prompts.md` — 図解プロンプト集

**GitHub issue自動記録**
```bash
gh issue create --repo dejiinaworks-png/note-skill \
  --title "[投稿記録] [記事タイトル]" \
  --body "## 記事情報\n- タイトル: \n- 文字数: \n- バズスコア: /100\n- 図解枚数: \n\n## 改善メモ\n"
```

**投稿チェックリスト出力**
```
□ タイトルにキーワードが入っている
□ アイキャッチバナーを設定した
□ 図解が3枚以上挿入されている
□ まとめにCTAがある
□ ハッシュタグを3〜5個設定した
□ 有料設定の検討（途中まで無料）
□ SNS（X）への告知投稿を準備した
```

## パラメータ仕様

| パラメータ | 必須 | デフォルト | 説明 |
|-----------|------|----------|------|
| テーマ | ✅ | - | 記事のメインテーマ |
| ターゲット | - | "noteユーザー全般" | 読者ペルソナ |
| 文体 | - | "語りかけ調" | ですます/だである/語りかけ調 |
| 文字数 | - | "3000" | 目標文字数 |
| 図解数 | - | "4" | 生成する図解の数 |
| カラー | - | "ブルー系" | 図解・バナーのテーマカラー |

```
output/
├── articles/
│   ├── YYYYMMDD_タイトル.md        # 完成記事

└── images/
    ├── YYYYMMDD_banner.png          # バナー（NanoBanana2で生成）
    ├── YYYYMMDD_fig_01.png          # フロー図
    ├── YYYYMMDD_fig_02.png          # 比較図
    └── YYYYMMDD_fig_03.png          # まとめ図
```
